# 架构设计文档

## 概述

OpenClaw Multi-Instance 方案允许在一台物理机上运行多个完全独立的 OpenClaw 实例，每个实例拥有独立的配置、数据、网络和身份。

## 设计目标

1. **完全隔离** - 每个实例独立运行，互不干扰
2. **易于管理** - 统一的管理脚本和命令
3. **灵活配置** - 支持不同的角色、性格和技能
4. **资源可控** - 独立的资源限制和监控
5. **数据安全** - 独立的数据存储和备份机制

## 组件说明

### 1. 实例生成器 (generate-instance.sh)

负责创建新的 OpenClaw 实例：
- 生成唯一的实例ID和端口
- 创建目录结构
- 生成个性化配置文件
- 注册到全局配置

### 2. 实例管理器 (manage-instances.sh)

负责管理实例生命周期：
- 启动/停止/重启实例
- 查看状态和日志
- 备份和恢复数据
- 验证配置完整性

### 3. Docker Compose 编排

定义多实例部署：
- 每个实例独立的服务定义
- 独立的网络命名空间
- 共享的扩展目录（可选）
- 资源限制和健康检查

### 4. 配置模板

预定义的角色模板：
- **管家** - 高效干练，注重效率
- **助手** - 友好热情，善于倾听
- **专家** - 严谨专业，追求精确
- **创意** - 富有创意，思维活跃

## 数据流

```
用户消息 → Channel → Gateway → Agent → 工具执行 → 响应
                ↑
                └── 每个实例独立的处理流程
```

## 安全设计

1. **网络隔离** - 每个实例独立的 Docker 网络
2. **数据隔离** - 独立的卷挂载
3. **配置隔离** - 独立的 API Keys 和凭证
4. **资源限制** - 可配置 CPU/内存限制

## 扩展性

- 支持动态添加新实例
- 支持自定义角色模板
- 支持共享扩展目录
- 支持自定义 Docker 镜像
